# Favicons sourced from Chrome `_favicon/` only

The popup's sidebar tabs show a favicon per Site. We use Chrome's MV3 `_favicon/` API (requires the `"favicon"` permission). We deliberately do **not** fall back to a remote service like Google's s2 or DuckDuckGo's icons endpoint, because every popup open would leak the user's manhwa-site list to a third party. For sites not in the user's Chrome history, the API returns its own generic placeholder served with HTTP 200 — we accept that as the visible state. We considered detecting the placeholder client-side to swap in a letter avatar, but Chrome serves it as a normal image response (no error), so `<img onerror>` and reka-ui's `<AvatarImage>` fallback never fire. Gating with `chrome.history.search` would require the `history` permission, which is too invasive for cosmetic gain.
